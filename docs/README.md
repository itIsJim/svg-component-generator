# Screenshots

The main README references three images from this folder. Capture them from the
running app (`npm run dev`, then load the sample with **Load sample**) and drop
them here with these exact names:

| File | What to capture | How |
| --- | --- | --- |
| `screenshot-code.png` | Full app window, **Code** tab active, React × Tailwind output visible | Light mode |
| `screenshot-preview.png` | **Live preview** tab with a layer hovered (tooltip + highlight visible), checkerboard background | Light mode |
| `screenshot-dark.png` | Same as the code view but in dark mode (toggle in the header) | Dark mode |

Tips for clean captures on macOS:

- Size the browser window to ~1400×900 before capturing.
- `Cmd+Shift+4`, then `Space`, then click the window — captures the window with
  its shadow on a transparent background (looks good on both GitHub themes).
- Keep file sizes reasonable (< 500 KB each); PNG is fine at this size.

Until the images exist, the README shows broken-image placeholders — add these
before sharing the repo link around.
